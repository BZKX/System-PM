# CONSENSUS_init

## 1. 需求描述与验收标准

### 1.1 核心需求
构建一个 **系统PM管理台**，用于解决多上线计划下的系统资源冲突问题。

### 1.2 关键特性
1.  **计划管理**: 录入上线计划（包含测试、内灰、外灰、全网四个阶段的时间节点）。
2.  **资源关联**: 每个计划可关联多个前后端系统。
3.  **冲突检测**:
    *   **规则**: 在 **外灰** 及 **全网** 期间，同一系统不可被多个计划同时占用。
    *   **表现**: 输入时间时实时提示冲突；时间轴视图中高亮显示冲突区域。
4.  **可视化**:
    *   提供 **系统维度** 的甘特图/时间线，直观展示各系统的被占用情况。
    *   提供 **计划卡片** 展示计划详情。

### 1.3 验收标准
*   [ ] 用户可以创建一个包含多个系统的上线计划。
*   [ ] 用户可以直观看到“外灰”和“全网”阶段的时间段。
*   [ ] 当用户尝试将两个计划的“外灰/全网”时间段重叠在同一个系统上时，系统应发出警告或标记红色。
*   [ ] 页面无报错，交互流畅。

## 2. 技术实现方案

### 2.1 技术栈
*   **前端框架**: React 18 + Vite + TypeScript
*   **UI组件库**: Ant Design (用于表单、布局、通用组件)
*   **样式库**: Tailwind CSS (用于排版和自定义样式)
*   **状态管理**: Zustand (管理计划数据、系统数据、冲突状态)
*   **时间处理**: Day.js
*   **图标库**: Lucide React

### 2.2 数据存储
*   **MVP阶段**: 使用浏览器 `localStorage` 进行数据持久化。

## 3. 边界限制
*   仅实现前端逻辑，无后端服务器。
*   不处理复杂的权限管理（假设单用户/PM使用）。
*   冲突检测仅基于时间段重叠，不涉及更复杂的资源调度算法。

## 4. 确认事项
*   所有不确定性已通过设定默认值（如技术栈选择）解决。
*   准备进入开发阶段。
