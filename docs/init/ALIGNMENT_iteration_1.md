# ALIGNMENT_iteration_1

## 1. 需求变更分析
用户在体验 MVP 版本后提出了三点改进意见，主要集中在视图丰富度、工具便捷性和UI美观度上。

## 2. 需求理解与确认

### 新增/变更需求
1.  **双维度仪表盘**:
    *   **现状**: 仅展示系统维度（Resource View）。
    *   **需求**: 需要支持 **系统维度** + **计划维度** 的切换展示。
    *   **价值**: "计划维度"能让PM直观看到每个计划的整体进度，而不仅是资源占用情况。
2.  **快速冲突检测工具 (Quick Check)**:
    *   **现状**: 只有在创建/编辑计划时触发冲突检测。
    *   **需求**: 独立的查询工具。输入 [外灰时间] + [系统列表] -> 输出 [是否冲突] + [可视化结果]。
    *   **场景**: PM 应对临时咨询，无需创建草稿计划即可快速答复。
3.  **系统管理卡片化**:
    *   **现状**: 列表 (Table) 风格。
    *   **需求**: 卡片 (Card) 风格展示。
    *   **价值**: 提升视觉体验，更符合"系统"这一实体的展示直觉。

## 3. 智能决策策略
*   **仪表盘切换**: 在 TimelineView 顶部增加 Segmented Control (分段控制器) 切换 "系统视图" / "计划视图"。
*   **快速检测**: 在仪表盘侧边或顶部增加一个显著的 "冲突预演" 或 "快速查询" 区域。考虑到易用性，建议使用 **Modal** 或 **Drawer** 承载这个临时查询功能，避免占据主屏过多空间。或者直接在 Dashboard 顶部作为一个折叠面板。
    *   *决策*: 放在 Dashboard 顶部作为一个 Card，默认折叠或展开，称为 "排期冲突速查"。
*   **卡片设计**: 系统卡片应包含：名称、类型（前后端图标区分）、归属部门、以及简单的状态（如"当前被占用" - 可选）。

## 4. 最终共识
*   **交付物**: 更新后的 Dashboard (支持视图切换 + 速查工具)，更新后的 System Management 页。
